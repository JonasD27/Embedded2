Teile einer Quellcodedatei

  • dontinclude filename
    • Includiert die Datei aber ohne Sie anzuzeigen
  • skip pattern
    • Überspringt alle Zeilen, bis eine Zeile das Pattern enthält
  • skipline pattern
    • wie skip nur dass die gefundene Zeile gedruckt wird
  • until pattern
    • druckt alle folgenden Zeilen, einschließlich der bei der das Pattern gefunden wurde
  • line pattern
    • unterdrückt folgende Leerzeilen bis zur nächsten nicht leeren Zeile, fallse diese das Pattern enthät wird diese gedruckt
@dontinclude doxy_exam1.c
@skip int16_t main(void)
@until } /* main()
int16_t main(void) {
return func(1,5);
} /* main() */

Teile einer Quellcodedatei - Snippet

Includiert alles zwischen (genau) zwei identischen Doxygen Markern in einer Quellcode Datei. Die Marker werden in [...] angegeben.

Überschriften formatieren

Diverse Möglichkeiten von Kapitel / Überschriften zu definieren

Kapitel mit den Doxygen Tags section subsection subsubsection erzeugt

@Section chap1 Überschrift (1te Ebene)
@section chap2 Überschrift (1te Ebene)
@subsection chap21 Überschrift (2te Ebene)
@subsubsection chap211 Überschrift (3te Ebene)

Kapitel Überschriften mit === bzw — in der nächsten Zeile erzeugt

    Überschrift (1te Ebene)  {#Ueb1}
    ================

    Überschrift (2te Ebene)  {#Ueb12}
    ---------------------

Überschriften mit #, ##, ###, #### erzeugt (nicht im Inhaltsverzeichnis)

    # Überschrift (1te Ebene)
    ## Überschrift (2te Ebene)
    ## Überschrift (2te Ebene)
    ### Überschrift (3te Ebene)
    #### Überschrift (4te Ebene)

Überschriften mit HTML Tags erzeugt

    <h1>Überschrift (1te Ebene) (html tag h1)</h1>
    <h2>Überschrift (2te Ebene) (html tag h2)</h2>
    <h3>Überschrift (3te Ebene) (html tag h3)</h3>
    <h4>Überschrift (4te Ebene) (html tag h4)</h4>

Doxygen Kommentare

Dieser Text könnte auch in einer C/H Datei stehen, sowie auch jeglicher doxygen Text in eigenständigen Dateien stehen könnte und Bezug auf die Codefragmente nimmt.

Es gibt verschiedene Arten Doxygen Kommentare zu schreiben.

  • In den Code Dateien wurde 3x / verwendet da dies den Vorteil hat größe Quellcodevereiche mittels Blockkommentaren auszukommentieren
  • hier wurde mit Blockkommentaren gearbeitet von / * * bis * / für jeweils einen Block, damit nicht in jeder Zeile 3x / geschrieben werden muss
  • in *.markup Dateien die nur für Dokumentationszwecke gedacht sind, werden keine einleitenden Kommentare benötigt. Natürlich wird aber auch kein Programmcode darin geparst!
  • noop weisst Doxygen an den Rest der Zeile zu ignorieren
    • gut für Kommentare oder Trennerzeilen in der Doku
    • gut geeignet zur Auskommentierung von Kommandos wie bug todo ...

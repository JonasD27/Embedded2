Doxykommentare

  • Vor dem Element
    • Blockkommentare /** ... */ oder /*! ...*/
      • * als erstes Zeichen einer Zeile wird ignoriert
      • Mit Banner aktiv auch: /*********************************** ... */
    • Zeilenkommentare /// oder //!
  • In der Zeile hinter dem Element
    • /**< ... */ oder /*!< — */
    • ///< oder //!<
    • VHDL: –!
  • @ oder \ leitet ein Doxy Command ein
    • @brief kurzer Beschreibungstext

Diverse gesammelte Hints

Doxygen Kommandos mit @noop auskommentieren

Mit dem Kommando @noop wird der Rest der Zeile von Doxygen ignoriert

  • Für Auskommentierung von Doxygen Code
  • Gut geeignet für Tags wie @bug, @todo, ... auszukommentieren, damit diese nicht im jeweiligen Index erscheinen. Auch wenn kein beschreibender Text hinterlegt ist, werden diese dennoch im Index mit aufgenommen.

Input Dateiformate

  • MPLABX IDE: ANSI / ISO-8859-1
  • Quartus: UTF-8

Bei falschem Dateiformat (Encoding) werden z.B. Umlaute falsch dargestellt.

  • Input Codierung im Doxyfile entsprechend der Entwicklungsumgebung einstellen
  • Falls es Probleme mit einer Datei gibt kann z.B. mit Notepad++ das Encoding Überprüft und ev. in das zum Projekt passende Encoding konvertiert werden. (z.B. diese Doxygen Dokumentationsseiten)
  • Erzeugt von 1.9.3
